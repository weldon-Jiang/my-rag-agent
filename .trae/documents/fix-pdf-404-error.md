# 修复知识库 PDF 文件 404 错误计划

## 问题分析
知识库目录 `knowledge/` 中的 PDF 文件（如 `项目经理面试常遇问题.pdf`）无法访问，返回 404 错误。

**原因**：当前 `/api/files` 路由只提供了文件列表查询、上传和删除功能，**缺少文件下载/访问接口**。虽然文件已存储在 `knowledge/` 目录，但没有 API 端点来检索这些文件内容。

## 修复步骤

### 步骤 1: 在 files.js 中添加文件下载路由
在 `server/routes/files.js` 添加 `GET /:filename` 路由，处理文件下载请求：
- 根据文件名获取文件完整路径
- 检查文件是否存在
- 返回文件内容，设置正确的 Content-Type

### 步骤 2: 更新 index.js 确保 knowledge 目录可访问
确认 `server/index.js` 中的路由配置正确

### 步骤 3: 测试验证
- 启动服务器
- 访问 PDF 文件 URL 验证修复效果
