# AI 对话记忆功能计划

## 目标
实现会话上下文记忆功能，自动识别新问题与历史问题的关联性，将相关上下文一起带入 AI 思考过程。

---

## 一、需求分析

### 1.1 功能描述
- 用户在同一个会话中连续提问时，系统自动分析问题间的关联性
- 如果新问题与之前问题相关（如都涉及"地球"），将相关历史问题+回答一起带入 Prompt
- 如果新问题与之前问题无关，直接回答新问题

### 1.2 示例场景
```
用户提问1: "地球为什么是圆的"  → AI回答1
用户提问2: "地球自转的原因"    → 关联"地球"，一起带入 Prompt
用户提问3: "今天天气怎么样"    → 无关，直接回答
```

---

## 二、实现方案

### 2.1 前端改造 (app.js)

1. **修改消息存储结构**
   - 在消息对象中增加 `relevanceInfo` 字段
   - 记录哪些历史消息与当前问题相关

2. **发送请求时传递历史消息**
   - 修改 sendMessage 函数
   - 将相关历史消息（用户问题 + AI回答）一起发送给后端

### 2.2 后端改造 (chat.js)

1. **新增记忆分析 API**
   - 接收当前问题和历史消息列表
   - 使用关键词/语义分析判断关联性
   - 返回需要带入上下文的历史消息

2. **修改 buildSystemPrompt 函数**
   - 增加 `historyContext` 参数
   - 将相关历史消息格式化为上下文

### 2.3 关联性判断逻辑

**方案 A：关键词匹配（简单实现）**
- 提取当前问题中的关键词
- 与历史问题关键词对比
- 有重叠则认为相关

**方案 B：使用 LLM 判断（推荐）**
- 将最近 N 条历史消息和当前问题一起发送给 LLM
- LLM 判断哪些历史消息与当前问题相关
- 返回相关消息列表

## 四、接口设计

### 4.1 请求参数
```javascript
{
  query: "地球自转的原因",
  mode: "ai",
  model: "xxx",
  history: [
    { role: "user", content: "地球为什么是圆的" },
    { role: "assistant", content: "地球是圆的因为..." },
    { role: "user", content: "月球是怎么形成的" },
    { role: "assistant", content: "月球可能是..." }
  ]
}
```

### 4.2 响应处理
- 后端分析 `history` 中哪些消息与当前 `query` 相关
- 将相关消息格式化后带入 System Prompt
- 返回回答时，前端更新消息列表

## 五、验收标准

1. ✅ 同一会话中，连续提问相关问题时，AI 能结合前文回答
2. ✅ 无关问题不带入无关上下文
3. ✅ 会话刷新后历史保留
4. ✅ 性能不影响响应速度
