# 优化上下文传递 - 只传递用户输入的关键内容

## 问题
当前上下文包含完整的工具返回内容（如"阜宁（使用盐城数据）当前天气：..."），这些原始数据不应该直接传给大模型。

## 目标
上下文只传递用户输入的关键内容（如城市名），而不是完整的工具返回。

## 当前流程（问题）
```
用户: "今天阜宁天气怎么样"
  ↓
工具返回: "阜宁（使用盐城数据）当前天气：晴，26°C..."
  ↓
上下文包含完整返回内容 → 大模型收到的是原始工具输出
```

## 新流程
```
用户: "今天阜宁天气怎么样"
  ↓
提取关键信息: "阜宁"
  ↓
上下文: "用户询问阜宁的天气"
  ↓
大模型基于原始问题 + 提取的信息生成回答
```

## 实现步骤

### Step 1: 提取用户输入的关键信息
在 context 中只传递：
- 用户原始问题
- 提取的关键实体（地名等）
- 不传递完整的工具返回

### Step 2: 修改上下文构建逻辑
```javascript
// 不要这样：
context = "阜宁当前天气：晴，26°C"

// 应该这样：
context = "用户询问：阜宁天气"
```

### Step 3: 让大模型基于原始问题回答
大模型应该基于用户的原始问题和提取的信息来回答，而不是基于工具的原始输出。
