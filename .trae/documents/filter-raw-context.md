# 优化上下文传递 - 过滤工具原始输出

## 问题
当前上下文直接包含工具的原始输出（如"阜宁（使用盐城数据）当前天气：晴，26°C..."），这些格式化后的原始数据直接传给了大模型。

## 目标
1. 过滤掉系统命令/终端输入
2. 只存储用户原始输入
3. 工具返回的内容需要经过智能处理/整理后再传给大模型

## 解决方案

### 修改工具返回格式
工具返回的 textContent 应该已经是整理好的内容，但当前格式过于技术化。

### 优化 textContent 格式
```javascript
// 当前格式（过于技术化）：
"阜宁（使用盐城数据）当前天气：\n天气状况：多云（白天）\n温度：26°C..."

// 应该的格式（更人性化）：
"阜宁今天的天气是多云，气温26°C，体感温度28°C，湿度适中。"
```

### 修改步骤

### Step 1: 优化天气工具的 textContent
让天气返回更自然的描述，而不是技术数据列表

### Step 2: 优化位置工具的 textContent
返回更简洁的位置信息

### Step 3: 确保上下文只包含整理后的内容
不传递原始技术数据
