# 计划：重构改名和关系检测逻辑

## 需求

简化意图检测逻辑，让名称提取函数直接通过正则匹配来识别模式，不需要关键词预检。

## 解决方案

### 1. 智能体改名（bot rename）
模式列表：
- `你叫([^\s，。,]{1,10})` - 你叫小通
- `以后叫你([^\s，。,]{1,10})` - 以后叫你小通
- `你以后叫([^\s，。,]{1,10})` - 你以后叫小通
- `称呼你([^\s，。,]{1,10})` - 称呼你小通
- `称呼你叫([^\s，。,]{1,10})` - 称呼你叫小通
- `你就叫([^\s，。,]{1,10})` - 你就叫小通
- `以后我就叫([^\s，。,]{1,10})` - 以后我就叫小通
- `叫我([^\s，。,]{1,10})` - 叫我小通
- `名字是([^\s，。,]{1,10})` - 名字是小通
- `从现在起(?:你|我就)叫([^\s，。,]{1,10})` - 从现在起你叫小通

### 2. 用户改名（user rename）
模式列表：
- `我叫([^\s，。,]{1,10})` - 我叫小明
- `我是([^\s，。,]{1,10})` - 我是小明
- `我的名字是([^\s，。,]{1,10})` - 我的名字是小明
- `以后叫我([^\s，。,]{1,10})` - 以后叫我小明

### 3. 关系设定（relationship）
模式列表：
- `你叫我([^\s，。,]{1,10})` - 你叫我老板
- `以后叫我([^\s，。,]{1,10})` - 以后叫我老板
- `以后你叫我([^\s，。,]{1,10})` - 以后你叫我老板
- `你以后叫我([^\s，。,]{1,10})` - 你以后叫我老板
- `称呼我为([^\s，。,]{1,10})` - 称呼我为主人
- `称呼我([^\s，。,]{1,10})` - 称呼我为上帝
- `我是你的([^\s，。,]{1,10})` - 我是你的老板
- `我是你([^\s，。,]{1,10})` - 我是你主人

## 实现

修改 `server/routes/chat.js`：

1. 修改 `extractBotName()` - 使用上述所有模式
2. 修改 `extractUserName()` - 使用上述所有模式
3. 修改 `extractRelationship()` - 使用上述所有模式
4. 移除或简化 `isRenameIntent()`, `isRenameUserIntent()`, `isSetRelationshipIntent()`
5. 在检测时直接调用提取函数，如果返回非 null 则保存
