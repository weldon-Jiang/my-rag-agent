# Skill + Tool 手动调度实现计划

## 需求分析

当前问题：之前的 tool_calls 机制依赖模型原生支持，但 MiniMax 等模型不支持。

新方案：**手动工具调度**
- 不依赖模型的 tool_calls 功能
- 根据查询意图**主动判断**需要调用哪些工具
- 收集工具返回结果后**组装上下文**发给 LLM

## 工作流程

```
用户查询
    ↓
意图分析 → 判断需要哪些工具
    ↓
依次/并行调用工具 → 获取结果
    ↓
组装上下文 → 发送给 LLM
    ↓
返回最终回答
```

## 工具定义（已在 skills/index.js）

| 工具名 | 功能 | 触发关键词 |
|--------|------|-----------|
| search_knowledge_base | 搜索知识库 | 搜索、查询、找 |
| recognize_image | OCR图片识别 | 图片、照片、截图 |
| extract_pdf_text | PDF文本提取 | PDF、文档 |
| analyze_video | 视频内容分析 | 视频、录像 |

## 实施步骤

### 步骤 1: 增强意图分析
在 chat.js 添加 `analyzeQueryIntents` 函数：
- 分析用户查询包含的意图
- 返回需要的工具列表

### 步骤 2: 实现工具调度器
添加 `dispatchTools` 函数：
- 根据意图调用对应工具
- 收集所有工具结果

### 步骤 3: 修改 chat.js 主流程
修改 `/api/chat` 路由：
- 先调用工具获取结果
- 将结果组装为上下文
- 发送完整上下文给 LLM

### 步骤 4: 测试验证
- 测试意图分析准确性
- 测试工具调用正确性
- 测试最终回答质量

## 优势

1. **兼容所有模型**：不依赖 tool_calls
2. **可控性强**：后端自主决定调用哪些工具
3. **调试简单**：工具调用流程清晰
4. **性能优化**：可以并行调用独立工具
