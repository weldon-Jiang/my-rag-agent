# 本地知识库智能体

一个功能完整的本地知识库智能体项目，支持与AI对话、文件上传、知识库检索和多种模型切换。

## 功能特性

- 💬 **AI对话**: 支持与多种AI模型对话
- 📚 **知识库管理**: 上传、删除本地文件到知识库
- 🔍 **智能检索**: 三种工作模式
  - **仅知识库**: 只检索本地知识库，没有就提示查不到
  - **仅AI模式**: 不检索知识库，直接使用AI回答
  - **混合模式**: 优先检索知识库，没有再使用AI
- 🤖 **多种模型**: 支持OpenAI和Ollama模型，可自由添加
- 🎨 **美观界面**: 现代化的用户界面设计

## 安装步骤

1. 安装依赖：
```bash
npm install
```

2. 配置环境变量：
编辑 `.env` 文件，填入你的API密钥：
```
PORT=3000
OPENAI_API_KEY=你的OpenAI_API密钥
OLLAMA_BASE_URL=http://localhost:11434
```

## 启动项目

```bash
npm start
```

然后在浏览器中访问 `http://localhost:3000`

## 使用说明

### 1. 上传文件到知识库
- 在左侧"知识库管理"区域选择文件
- 点击"上传文件"按钮
- 文件会自动保存到 `./knowledge` 目录下

### 2. 选择工作模式
- **仅知识库**: 只在本地文件中搜索答案
- **仅AI模式**: 直接调用AI模型回答
- **混合模式**: 优先使用知识库，没有则使用AI

### 3. 切换模型
- 从下拉列表中选择可用的模型
- 可以添加自定义模型（需在后端配置对应API）

### 4. 开始对话
- 在输入框中输入问题
- 按 Enter 或点击"发送"按钮

## 支持的模型

### OpenAI模型（需要API密钥）
- GPT-3.5 Turbo
- GPT-4

### Ollama模型（需要本地安装Ollama）
- Llama 2
- Mistral

## 添加自定义模型

1. 在前端界面的"添加新模型"区域填写：
   - 模型ID（如：openai-gpt-4o）
   - 模型名称（如：GPT-4o）
   - 供应商（如：OpenAI）

2. 在后端 `server/routes/chat.js` 中添加相应的API调用逻辑
